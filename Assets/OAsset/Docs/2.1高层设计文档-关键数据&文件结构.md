## 3. 关键数据结构

### 3.1 AssetInfo（资源条目）

描述一个可加载资源在清单中的记录。

```csharp
[Serializable]
public class AssetInfo
{
    public string AssetPath;       // 资源完整路径，如 "Assets/ToAB/Player.prefab"
    public int BundleID;           // 所属 Bundle 在 BundleList 中的索引
}
```

### 3.2 BundleInfo（Bundle 条目）

描述一个 AssetBundle 的元信息。

```csharp
[Serializable]
public class BundleInfo
{
    public string BundleName;      // Bundle 文件名，如 "assets_toab_triangle.bundle"
    public uint UnityCRC;          // Unity 构建时生成的 CRC，用于加载校验
    public string FileHash;        // 文件 MD5 哈希，作为唯一标识和缓存键
    public uint FileCRC;           // CRC32，用于下载完整性校验
    public long FileSize;          // 文件大小（字节）
    public int[] DependBundleIDs;  // 依赖 Bundle 的索引数组
}
```

### 3.3 Manifest（资源清单）

管理所有 AssetInfo 和 BundleInfo 的容器，提供按路径查询能力。

```csharp
[Serializable]
public class Manifest
{
    public List<AssetInfo> AssetList;                 // 所有资源条目
    public List<BundleInfo> BundleList;               // 所有 Bundle 条目
    private Dictionary<string, AssetInfo> _assetLookup; // 路径→AssetInfo 查询字典（运行时构建）
}
```

**索引关系图**：

```
AssetList[i].BundleID ──────────────► BundleList[BundleID]
BundleList[i].DependBundleIDs[j] ──► BundleList[DependBundleIDs[j]]
```

加载资源时，通过 `AssetInfo.BundleID` 找到所属 `BundleInfo`，再从 `BundleInfo.DependBundleIDs` 获取依赖关系。所有 ID 字段均为 `BundleList` 数组的下标索引，而非字符串引用。

### 3.4 OAssetConfig（运行时配置）

```csharp
[CreateAssetMenu(fileName = "OAssetConfig", menuName = "Config/OAssetConfig")]
public class OAssetConfig : ScriptableObject
{
    public string serverUrl;            // 资源服务器地址
    public int timeoutSeconds = 30;     // 网络超时（秒）
    public string cacheRootPath;        // 缓存子目录（相对于 persistentDataPath）
}
```

### 3.5 运行时状态（OAsset 单例内部）

```
OAsset._manifest         : Manifest              ← 当前版本的资源清单
OAsset._loadedBundles    : Dict<string, AssetBundle>     ← 已加载的 Bundle（key=FileHash）
OAsset._loadingBundles   : Dict<string, UniTask<AssetBundle>> ← 正在加载的 Bundle（防重复）
OAsset._version          : string                ← 当前版本号
OAsset._cacheRoot        : string                ← 缓存根目录绝对路径
```

## 4. 文件结构

### 4.1 源码文件结构

```
Assets/OAsset/
├── Runtime/
│   ├── OAsset.Runtime.asmdef
│   ├── OAsset.cs              ← 运行时主管理器（单例）
│   ├── OAssetConfig.cs        ← 配置 ScriptableObject
│   ├── Manifest.cs            ← Manifest / AssetInfo / BundleInfo 数据结构
│   └── FileDownloader.cs      ← 文件下载、CRC32 校验、缓存 IO
├── Editor/
│   ├── OAsset.Editor.asmdef
│   ├── OAssetBuilder.cs       ← 打包工具（菜单命令）
│   ├── IAssetBundleStrategy.cs ← 打包策略接口
│   └── DefaultStrategy.cs     ← 默认打包策略
└── Docs/                      ← 文档目录
```

### 4.2 构建输出文件结构

构建后位于 `Assets/StreamingAssets/AB/`：

```
Assets/StreamingAssets/AB/
├── version.txt                              ← 版本号，如 "2026.3.15.043225"
├── 2026.3.15.043225.manifest.json           ← 资源清单 JSON
├── assets_toab_triangle.bundle              ← Bundle 文件
├── assets_toab_textures_avatar.bundle       ← Bundle 文件
└── ...
```

### 4.3 运行时缓存目录结构

运行时位于 `Application.persistentDataPath/<cacheRootPath>/`：

```
<缓存根目录>/
├── version.txt                              ← 当前版本号
├── <版本号>.manifest.json                    ← 当前版本清单
└── BundleFiles/
    ├── e9/                                  ← MD5 前 2 位（分散子目录）
    │   └── e99a18c428cb38d5f260853678922e03/ ← 完整 MD5（Bundle 唯一标识）
    │       ├── __data                       ← Bundle 二进制数据
    │       └── __info                       ← 校验信息："CRC32,FileSize"
    ├── 33/
    │   └── 33558d69e5e272ee75f7306cfac06164/
    │       ├── __data
    │       └── __info
    └── ...
```

**缓存设计要点**：
- 以 MD5（FileHash）而非文件名作为目录名，使得同名但内容不同的 Bundle 可以共存
- MD5 前 2 位作为一级子目录，避免单目录下文件过多
- `__info` 文件格式为纯文本 `{CRC32},{FileSize}`，用于启动时快速校验

### 4.4 清单文件格式（manifest.json）

文件命名：`<版本号>.manifest.json`，版本号格式 `yyyy.M.d.HHmmss`。

```json
{
    "AssetList": [
        {
            "AssetPath": "Assets/ToAb/Triangle.prefab",
            "BundleID": 0
        }
    ],
    "BundleList": [
        {
            "BundleName": "assets_toab_triangle.bundle",
            "UnityCRC": 2507661759,
            "FileHash": "33558d69e5e272ee75f7306cfac06164",
            "FileCRC": 1274731402,
            "FileSize": 34011,
            "DependBundleIDs": []
        }
    ]
}
```

### 4.5 版本号文件（version.txt）

纯文本文件，内容为一行版本号字符串：

```
2026.3.15.043225
```

格式说明：`年.月.日.时分秒`（24 小时制），由打包时间自动生成。

## 5. 打包策略接口

```csharp
public interface IAssetBundleStrategy
{
    string GetBundleName(string assetPath);   // 资源路径 → Bundle 名称
    string[] GetScanDirectories();             // 需扫描的目录列表
    string[] GetAssetFilters();                // 允许的文件扩展名列表
}
```

**默认策略命名规则**：

```
输入: Assets/ToAB/Prefabs/Player.prefab
  → 小写:     assets/toab/prefabs/player.prefab
  → 去扩展名:  assets/toab/prefabs/player
  → / 转 _:   assets_toab_prefabs_player
  → 加后缀:   assets_toab_prefabs_player.bundle
```

## 6. 校验体系

系统使用多层校验确保数据完整性：

| 校验手段 | 生成时机 | 使用时机 | 用途 |
|----------|----------|----------|------|
| `FileHash`（MD5） | 构建时计算 | 运行时缓存寻址 | 唯一标识 Bundle 版本，作为缓存目录名 |
| `FileCRC`（CRC32） | 构建时计算 | 下载后校验、首次复制时校验 | 验证文件内容完整性 |
| `FileSize` | 构建时记录 | 下载后校验、首次复制时校验 | 快速检测文件截断或异常 |
| `UnityCRC` | Unity 构建自动生成 | 加载时可选校验 | Unity AssetBundle 内部校验 |
