# 1.1 需求文档

## 1. 项目背景

在 Unity 游戏项目中，资源管理是核心基础设施之一。市面上的 YooAsset 等方案功能完善但过于复杂，对中小型项目而言存在理解成本高、接入复杂的问题。OAsset 旨在提供一个精简、易理解、可控的 AssetBundle 资源管理方案。

## 2. 项目目标

打造一个精简版的资源管理系统，覆盖 AssetBundle 的**打包、加载、缓存、热更新**核心流程。

### 2.1 与 YooAsset 的简化对比

| 简化项 | YooAsset | OAsset |
|--------|----------|--------|
| Package 概念 | 支持多 Package | 移除，仅单一 Package |
| 资源寻址 | 支持别名、短路径等多种寻址方式 | 仅支持文件完整路径寻址（如 `Assets/ToAB/Player.prefab`） |
| 引用计数 | 完整的引用计数与自动卸载 | 暂不实现 |
| 下载管线 | 并发下载、断点续传、限速 | 仅顺序下载、失败重试 |
| 加密 | 支持 Bundle 加密 | 暂不实现 |

## 3. 项目依赖

| 依赖 | 用途 |
|------|------|
| UniTask | 提供高性能异步操作支持，替代 Unity 原生协程 |

## 4. 功能需求

### FR-1 AssetBundle 打包

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-1.1 | 提供可扩展的打包策略接口，支持自定义分包规则 | P0 |
| FR-1.2 | 提供默认打包策略：按文件路径命名，一个文件对应一个 Bundle | P0 |
| FR-1.3 | 扫描指定目录，自动发现需要打包的资源 | P0 |
| FR-1.4 | 使用 LZ4 压缩格式构建 AssetBundle | P0 |
| FR-1.5 | 支持增量构建和强制重建两种模式 | P1 |
| FR-1.6 | 构建完成后自动生成资源清单文件（manifest.json） | P0 |
| FR-1.7 | 自动生成版本号文件（version.txt），格式为打包时间（精确到秒） | P0 |
| FR-1.8 | 提供 Unity 菜单入口操作打包相关功能 | P1 |

### FR-2 运行时初始化

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-2.1 | 首次启动时，将 StreamingAssets 内置资源复制到缓存目录，复制时校验文件大小和 CRC32 | P0 |
| FR-2.2 | 非首次启动时，检查服务器版本号，判断是否需要热更新 | P0 |
| FR-2.3 | 版本不一致时，下载最新清单文件并更新本地，同时清理旧清单文件 | P0 |
| FR-2.4 | 解析清单文件，建立资源路径到 Bundle 的映射关系 | P0 |

### FR-3 资源加载

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-3.1 | 提供泛型异步加载接口 `LoadAssetAsync<T>(string assetPath)` | P0 |
| FR-3.2 | 自动处理 Bundle 依赖（从 BundleInfo.DependBundleIDs 获取），加载资源前先加载其依赖的 Bundle | P0 |
| FR-3.3 | 编辑器模式下直接使用 `AssetDatabase` 加载，无需打包 | P1 |
| FR-3.4 | 已加载的 Bundle 进行内存缓存，避免重复加载 | P0 |
| FR-3.5 | 并发请求同一 Bundle 时，复用同一个加载任务（使用 UniTask.Lazy 消除竞态窗口） | P0 |

### FR-4 资源下载

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-4.1 | 从服务器下载 Bundle 文件 | P0 |
| FR-4.2 | 下载失败时自动重试，最多 3 次，指数退避（通过通用重试方法实现） | P0 |
| FR-4.3 | 下载完成后校验文件大小和 CRC32 | P0 |
| FR-4.4 | 校验通过后按缓存格式写入磁盘 | P0 |

### FR-5 资源卸载

| 编号 | 需求 | 优先级 |
|------|------|--------|
| FR-5.1 | 支持按 Bundle 哈希卸载单个 Bundle | P0 |
| FR-5.2 | 支持一次性卸载所有已加载的 Bundle | P0 |

## 5. 非功能需求

| 编号 | 需求 | 说明 |
|------|------|------|
| NFR-1 | 跨平台 | 支持 Windows Standalone 和 Android（StreamingAssets 读取方式不同） |
| NFR-2 | 异步非阻塞 | 所有 IO 和网络操作使用 UniTask 异步执行，不阻塞主线程 |
| NFR-3 | 可配置 | 服务器地址、超时时间、缓存路径等通过 ScriptableObject 配置 |
| NFR-4 | 数据完整性 | 下载和首次复制均使用 CRC32 + 文件大小双重校验 |

## 6. 已知限制与后续规划

| 限制 | 描述 | 规划 |
|------|------|------|
| 无引用计数 | 已加载的 Bundle 不会自动卸载，随游玩不断增多 | 后续版本实现 |
| 无断点续传 | 下载中断需从头重新下载 | 后续版本扩展 |
| 无并发下载 | 一次只下载一个文件 | 后续版本扩展 |
| 无资源加密 | Bundle 文件无加密保护 | 后续版本添加 |
