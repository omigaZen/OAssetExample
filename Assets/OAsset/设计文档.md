# 设计文档

## 1. 概述

### 1.1 项目目标

本项目旨在打造一个精简版的YooAsset资源管理系统。相比YooAsset，做了以下简化：

| 简化项 | 说明 |
|--------|------|
| Package概念 | 移除，仅需单一Package |
| 寻址名 | 移除，直接用文件路径（如 `Assets/ToAB/Player.prefab`）寻址 |
| 琐碎功能 | 移除非核心功能 |

具体实现可参考YooAsset使用的API。

### 1.2 项目依赖
- UniTask

### 1.3 系统说明

本文档描述了游戏中资源管理系统的功能需求和技术规范。系统基于Unity的AssetBundle机制，实现资源的分包、打包、加载、缓存和卸载功能，支持热更新场景。

### 1.4 模块说明

| 模块 | 位置 | 职责 |
|------|------|------|
| OAsset | 运行时 | 资源加载、缓存、卸载 |
| OAssetBuilder | 编辑器 | AssetBundle打包、清单生成 |
| IAssetBundleStrategy | 编辑器 | 打包策略定义 |
| DefaultStrategy | 编辑器 | 默认打包策略实现 |
| OAssetConfig | 运行时 | 配置管理 |
| FileDownloader | 运行时 | 文件下载 |

### 2.1 打包配置（AssetBundleStrategy）

#### 2.1.1 打包策略接口 (IAssetBundleStrategy)

**需求描述**：定义可扩展的打包策略接口，支持不同的分包规则。

**接口定义**：

```csharp
public interface IAssetBundleStrategy
{
    // 根据资源路径获取所属Bundle名称
    string GetBundleName(string assetPath);

    // 获取需要扫描的资源目录
    string[] GetScanDirectories();

    // 获取需要打包的资源类型过滤器
    string[] GetAssetFilters();
}
```


#### 2.1.2 默认打包策略 (DefaultStrategy)

**需求描述**：提供默认的分包策略，按目录结构划分Bundle。

**功能规格**：

| 功能   | 描述                                                          |
| ---- | ----------------------------------------------------------- |
| 分包规则 | 按文件路径生成Bundle名称，格式：`路径_文件名.bundle`（小写，`/` 转 `_`）            |
| 扫描目录 | 默认扫描 `Assets/ToAB` 目录                                       |
| 资源过滤 | 支持 `.prefab`, `.png`, `.jpg`, `.mat`, `.anim`, `.asset` 等类型 |

**命名规则**：

```
路径转换规则：
1. 转为小写
2. 将路径分隔符 / 替换为 _
3. 去掉文件扩展名
4. 添加 .bundle 后缀
```

**命名规则示例**：

```
Assets/ToAB/Prefabs/Player.prefab     → assets_toab_prefabs_player.bundle
Assets/ToAB/Textures/Avatar.png       → assets_toab_textures_avatar.bundle
Assets/ToAB/Configs/GameConfig.bytes  → assets_toab_configs_gameconfig.bundle
Assets/ToAB/Scenes/MainScene.unity    → assets_toab_scenes_mainscene.bundle
```


### 2.2 打包结果文件列表

#### 2.2.1 输出文件

打包完成后生成的文件结构：

| 文件 | 说明 |
|------|------|
| `version.txt` | 版本号文件，版本号为打包时间，格式如：`2026.3.4.093025` |
| `<版本号>.manifest.json` | 资产清单文件，存储资产和AB包的依赖信息及包的校验信息 |
| 若干AB文件 | AssetBundle资源文件 |

| 路径                                       | 描述         |
| ---------------------------------------- | ---------- |
| `Assets/StreamingAssets/AB/`             | Bundle输出目录 |

| `Assets/StreamingAssets/AB/version.txt`  | 版本文件       |


#### 2.2.2 清单文件格式
文件名：<版本号>.manifest.json
数据结构：
- AssetList.item
	- AssetPath：资源文件路径
	- BundleID：资源所属包（对应BundleList数组下标）
- BundleList.item
	- BundleName: 包名
	- UnityCRC：构建时由 Unity 自动生成，用于Unity AssetBundle 加载校验
	- FileHash：包md5 用于唯一标识包
	- FileCRC：和FileSize一起用于文件完整性校验
	- FileSize：文件长度
	- DependBundleIDs：依赖的包（对应BundleList数组下标）

**说明**：BundleID 和 DependBundleIDs 的值为整数，对应 BundleList 数组的下标索引。加载资源时，通过 AssetInfo.BundleID 找到所属 BundleInfo，再从 BundleInfo.DependBundleIDs 获取依赖关系。

### 2.4 资源文件存储路径

#### 2.4.1 内置资源目录 (StreamingAssets)

打包后随客户端分发的资源存放在 StreamingAssets 目录：

```
StreamingAssets/AB/
├── version.txt                         ← 版本号文件
├── 2026.3.4.093025.manifest.json       ← 资源清单文件
├── assets_toab_prefabs_player.bundle   ← Bundle文件
├── assets_toab_prefabs_enemy.bundle
├── assets_toab_shaders.bundle
└── ...
```

#### 2.4.2 缓存目录 (cacheRootPath)

运行时缓存使用MD5作为文件夹名称存储，结构如下：

```
缓存根目录(Application.persistentDataPath/xxx)/
└── version.txt
└── <version>.manifest.json
└── BundleFiles/
    └── e9/                              ← MD5前2位 (用于分散文件)
        └── e99a18c428cb38d5f260853678922e03/  ← 完整MD5 (BundleGUID)
            ├── __data                   ← Bundle数据文件
            └── __info                   ← 校验信息文件 (CRC32 + Size)
```

**__info 文件格式**：

文本格式，内容为 `{CRC32},{FileSize}`，例如：
```
2845673215,24586
```

**说明**：
- 首次启动时，StreamingAssets 中的资源会被复制到缓存目录，复制过程中会校验每个 Bundle 的文件大小和 CRC32
- 后续热更新下载的资源直接写入缓存目录
- 加载资源时统一从缓存目录读取

## 4. 运行时过程

### 4.1 初始化

**需求描述**：游戏启动时加载资源清单，初始化资源映射表。

**初始化流程**：

```
1. 检查 cacheRootPath 目录是否存在
   - 不存在：首次启动，执行首次启动初始化流程（见下方）
   - 存在：跳到步骤2

2. 下载 version.txt 文件判断版本
3. 如果版本与本地不一致：
   a. 更新本地版本号
   b. 下载对应版本的清单文件
   c. 删除旧版本的清单文件
4. 清单文件解析到内存中
   a. 建立资源路径AssetPath与AB包的对应关系
   b. 建立AB包信息（校验信息、包之间依赖关系等）
```

#### 4.1.1 首次启动初始化

**需求描述**：客户端安装后首次启动时，将 StreamingAssets 中的内置资源复制到缓存目录。

**首次启动流程**：

```
1. 创建 cacheRootPath 目录结构
   ├── 创建 BundleFiles/ 目录

2. 读取 StreamingAssets/AB/version.txt 获取内置版本号

3. 读取 StreamingAssets/AB/<版本号>.manifest.json 清单文件

4. 遍历清单中的每个Bundle：
   a. 从 StreamingAssets/AB/<BundleName> 读取Bundle文件
   b. 校验文件大小和 CRC32（与清单中的值比对）
   c. 使用清单中的 FileHash（即MD5）作为 BundleGUID
   d. 按缓存规则写入cacheRootPath：
      - 创建目录: BundleFiles/{MD5前2位}/{MD5}/
      - 写入文件: __data (Bundle数据)
      - 写入文件: __info (CRC32 + Size，从清单获取)

5. 复制版本文件和清单文件到 cacheRootPath：
   - version.txt
   - <版本号>.manifest.json

6. 标记初始化完成
```

**实现要点**：

- StreamingAssets 在不同平台的访问方式不同，需使用 UnityWebRequest 或 AssetBundle.LoadFromFile
- Windows/Standalone：可直接通过文件路径访问
- Android：需通过 UnityWebRequest 从 APK 内读取
- 复制过程中需显示加载进度
- 复制过程中需校验每个 Bundle 的文件大小和 CRC32

**示例代码**：

```csharp
private async UniTask InitFromStreamingAssets()
{
    string streamingPath = Path.Combine(Application.streamingAssetsPath, "AB");
    string cachePath = _cacheRoot;

    // 1. 创建缓存目录
    Directory.CreateDirectory(Path.Combine(cachePath, "BundleFiles"));

    // 2. 读取内置版本号
    string versionUrl = GetStreamingAssetsUrl("version.txt");
    string version = await ReadStreamingFile(versionUrl);

    // 3. 读取清单文件
    string manifestUrl = GetStreamingAssetsUrl($"{version}.manifest.json");
    string manifestJson = await ReadStreamingFile(manifestUrl);
    var manifest = Manifest.FromJson(manifestJson);

    // 4. 复制每个Bundle到缓存目录（含校验）
    foreach (var bundle in manifest.BundleList)
    {
        string bundleUrl = GetStreamingAssetsUrl(bundle.BundleName);
        byte[] bundleData = await ReadStreamingFileBytes(bundleUrl);

        // 校验文件大小和 CRC32
        if (bundleData.Length != bundle.FileSize)
            throw new Exception($"[OAsset] StreamingAssets size mismatch: {bundle.BundleName}");
        uint crc = FileDownloader.ComputeCRC32(bundleData);
        if (crc != bundle.FileCRC)
            throw new Exception($"[OAsset] StreamingAssets CRC mismatch: {bundle.BundleName}");

        FileDownloader.WriteBundleToCache(bundleData, bundle, cachePath);
    }

    // 5. 复制版本和清单文件
    File.WriteAllText(Path.Combine(cachePath, "version.txt"), version);
    File.WriteAllText(Path.Combine(cachePath, $"{version}.manifest.json"), manifestJson);
}
```


### 4.2 加载资源

**需求描述**：提供异步加载接口，避免阻塞主线程。

**接口定义**：

```csharp
// 异步加载资源
public async UniTask<T> LoadAssetAsync<T>(string assetPath) where T : Object;
```


**加载流程**：

```
编辑器模式:
  → AssetDatabase.LoadAssetAtPath()

运行时模式:
  1. 根据assetPath找到对应的AB包信息
  2. 从 BundleInfo.DependBundleIDs 获取依赖列表
  3. 先加载依赖的AB包（递归执行本流程）
  4. 如果AB包已经加载 跳到8
  5. 检查是否正在加载中
     - 是：加入等待队列，加载完成后统一回调
     - 否：继续6
  6. 使用AB包MD5查找本地缓存是否存在
     - 存在：继续7
     - 不存在：下载Bundle到缓存目录（见下载流程）
  7. 加载对应AB包(AssetBundle.LoadFromFileAsync)
  8. 从AB包加载出资源(bundle.LoadAssetAsync<T>)
```

#### 4.2.1 Bundle缓存管理

**需求描述**：已加载的Bundle进行缓存，避免重复加载。

**缓存机制**：

```csharp
// Bundle缓存字典，key为Bundle的MD5（FileHash）
private Dictionary<string, AssetBundle> _loadedBundles;

// 检查缓存后再加载
if (_loadedBundles.TryGetValue(bundleMD5, out var bundle))
{
    return bundle;  // 返回缓存的Bundle
}
```

#### 4.2.2 下载资源

**需求描述**：提供基于UnityWebRequest的文件下载功能，支持版本检查和资源更新。

**接口定义**：

```csharp
// 下载单个文件
public IEnumerator DownloadFile(string url, string localPath, Action<float> onProgress, Action<bool> onComplete);

// 下载Bundle
public IEnumerator DownloadBundle(BundleInfo bundle, Action<float> onProgress, Action<bool> onComplete);

// 批量下载（获取所有需要更新的Bundle）
public IEnumerator DownloadUpdates(Action<float> onTotalProgress, Action<int, int> onBundleComplete, Action<bool> onComplete);
```

**下载流程**：

```
1. 计算总下载大小，提示用户（可选）
2. 逐个下载Bundle：
   a. 下载到临时路径 <BundleName>.tmp
   b. 校验文件完整性（检查文件大小 + 计算CRC32与清单比对）
   c. 校验失败：删除临时文件，重新下载（重试3次）
   d. 校验成功：按缓存格式写入
      - 创建目录: BundleFiles/{MD5前2位}/{MD5}/
      - 重命名文件: <BundleName>.tmp → __data
      - 写入校验信息: __info (CRC32 + Size)
```

**简单实现（当前版本）**：

```csharp
// 使用UnityWebRequest实现基础下载 localPath是AB包的名字（带后缀）
public IEnumerator DownloadFile(string url, string localPath, Action<float> onProgress, Action<bool> onComplete)
{
    using (UnityWebRequest request = UnityWebRequest.Get(url))
    {
        request.downloadHandler = new DownloadHandlerFile(localPath);
        var operation = request.SendWebRequest();

        while (!operation.isDone)
        {
            onProgress?.Invoke(operation.progress);
            yield return null;
        }

        if (request.result == UnityWebRequest.Result.Success)
        {
            onComplete?.Invoke(true);
        }
        else
        {
            Debug.LogError($"Download failed: {request.error}");
            onComplete?.Invoke(false);
        }
    }
}
```

**后续扩展点**：
- 断点续传：记录已下载字节数，支持从断点继续
- 并发下载：同时下载多个文件
- 下载限速：避免占用过多带宽

## 5. 错误处理

**需求描述**：定义各阶段的错误处理策略，确保系统健壮性。

**错误类型与处理策略**：

| 阶段  | 错误类型       | 处理策略             |
| --- | ---------- | ---------------- |
| 初始化 | 清单下载失败     | 重试3次，失败后使用本地缓存清单 |
| 初始化 | 清单解析失败     | 返回错误，记录日志，使用本地缓存清单 |
| 加载  | Bundle不存在  | 返回null，记录错误日志    |
| 加载  | Bundle加载失败 | 返回null，清除损坏的缓存文件 |
| 下载  | 网络超时       | 重试3次，指数退避        |
| 下载  | 文件校验失败     | 删除临时文件，重新下载      |
| 下载  | 磁盘空间不足     | 清理旧缓存，提示用户       |
